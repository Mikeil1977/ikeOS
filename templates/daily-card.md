# DailyCard Template

```markdown
### <id> - <title>

- `domain`: <work|home|health|admin_money|projects|system|personal>
- `why_now`: <why this matters today>
- `next_action`: <smallest useful next action>
- `owner`: <Mike|MikeOS|CLARA|PM|EA|SA|Home|Motion|customer|other>
- `urgency`: <now|today|soon|watch|blocked>
- `source_refs`: <canonical links or paths only>
- `freshness`: <fresh|today|stale|unknown|unavailable|timestamp>
- `action_mode`: <read_only|route_only|propose_write|approved_write>
```

## Rules

- Keep the title short enough to scan.
- Link to the owning system rather than copying source content.
- Use `blocked` only when a named condition prevents progress.
- Use `unavailable` when a source could not be checked.
